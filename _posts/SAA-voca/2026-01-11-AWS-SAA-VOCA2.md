- Scalability : The ability of a system, application, or infrastructure to handle increasing load (more users, more data, more traffic) by adding resources - without breaking or slowing down.

- Vertical Scalability : Increasing the power of a single server or resource by adding more CPU, memory (RAM), disk, or IOPS to handle more load.

- Horizontal : Adding more machines, servers, or instances to handle increased traffic or load.

- Elasticity : The ability of a system to automatically increase or decrease resources based on the current demand.

- Implies : Suggests or indicates something without saying it directly.

- Offerings : The products, services, or features that a company (like AWS) provides to customers.

- Passive : Something that does not actively do anything, but waits, observes, or responds only when needed.

- Expose : To make something visible or accessible to others - especially something that was hidden or private.

- Seamlessly : Doing something smoothly and without interruption,problems, or the user even noticing.

- Stickiness : Making sure a user always connects to the smae server or instance during a session - even when aa load balancer is used.

- Guarantees : A promise or assurance that something will happen in a certain way, with specific reliability or results.

- Knobs : Settings or configuration options that you can adjust (tune) to control how a system behaves.

- Configuration : The settings, options, and values that define how a system, application, or service behaves.
